GlassBox Sentinel

A runtime assurance layer for Mistral Vibe that turns agent incidents into verified, reusable immunity.

Runtime assurance, outside the agent loop

Let Mistral Vibe move fast. Let Sentinel make every failure safer next time.

GlassBox Sentinel receives a minimal, redacted stream of Vibe execution events. At a meaningful boundary, it contains the unsafe path, captures the smallest reproducible incident, compiles a bounded policy repair, and verifies both safety and task utility.

What requires human authorization in a live Vibe session?

Sentinel can autonomously contain by returning a wait or deny at the next configured Vibe tool boundary. It does not autonomously invent a new Vibe instruction or resume the session. A live recovery requires an accepted, auditable override or separately configured policy.

Privacy boundary

Workspace paths, credentials, raw prompts, and full transcripts are excluded before the event reaches Sentinel.

Why a separate assurance plane?

Orchestration can enforce immediate local behavior. Sentinel makes an incident reusable: it preserves cross-run evidence, a bounded repair, and an independent verification result.

Fixture versus live integration

The public proof is a deterministic isolated worktree. Live Vibe enforcement activates only when the checked-in hook and its authenticated token are configured.

Evidence over claims

Trust the boundary, the artifact, and the scope.

Sentinel is designed to show what it knows, what it did, and what it deliberately never receives.

What Sentinel verifies

  • The unsafe path matches the captured reproducer and is blocked.
  • The intended legitimate route still succeeds.
  • The repair is bounded, inspectable, and attached to evidence.

What Sentinel does not claim

  • It does not inspect full Vibe transcripts.
  • It does not receive workspace paths or credentials.
  • The public proof is fixture-driven; it is not a live Vibe session.

Hackathon scope

  • Runs are held in memory.
  • The worktree verifier is real but disposable.
  • Live Vibe enforcement requires configured hook credentials.

Why this matters

Agent hooks and local loops can stop one run. GlassBox Sentinel turns the incident into a durable, reviewable assurance artifact that can make the next run safer.
